Walkthrough

Development Procedure

Let's walk through the basic development process for a PhoneGap app.

In order to make and test your app, you'll have to:
  1. Install the PhoneGap CLI.
  2. Create a GitHub repo for the project (necessary for the build service).
  3. Create a blank PhoneGap app
  4. Set app descriptors, preferences, and plugins in config.xml
  5. Write your HTML, CSS, and JS.
  6. Test in the browser, or on PhoneGap's mobile developer app
  7. Push and build on the PhoneGap Build Service.

Install PhoneGap CLI

  • PhoneGap projects are written on a desktop through either the Desktop App or the CLI program
  • PhoneGap CLI requires node.js, and can be installed through npm:
    $ npm install -g phonegap@latest
  • All phonegap commands start with $ phonegap

Create a GitHub Repository

Make sure to make the GitHub repo public and initialized with a README.

Create a Blank App

Create a blank project in the GitHub repo with the follow CLI commmand:
$ phonegap create -n TestApp --template blank app

Project Structure

The basic structure of a PhoneGap project consists of a config.xml file (which contains the settings and properties of the app), a www folder, and often a merges folder. The www folder contains the core files of the app, and the merges folder contains platform specifc files that are meant to replace those in the www folder when needed. Though not shown in the diagram below, phonegap projects often also contain a plugins folder for the various PhoneGap plugins that can be used by an app.

Configure the App

  • The config.xml file is used by the PhoneGap CLI and the Build service to configure and compile the app
  • The file contains app descriptors, such as name, id, version, and author
  • It also contains app settings and preferences, such as access permissions, supported platforms, icons and splash screens, and included plugins
  • See the exact configuration instructions in the official PhoneGap docs.

Write HTML, CSS, and JS

Write the app's code in the same way you would create a regular web app, but optimized for mobile. Your code can be vanilla, or it can utilize popular frameworks like Bootstrap, React, or JQuery mobile. Plugin functionality can be incorporated in the app's Javascript files. Test on the Mobile Developer App

  • PhoneGap provides a free mobile developer app that can be used to preview your app.
  • Install and find more information about the app on the PhoneGap website.
  • Use the $ phonegap serve command to start the server and receive an IP address to enter in the mobile app.

Compile and Test on PhoneGap Build

  • Adobe provides a free cloud service for rapidly compiling and distributing PhoneGap apps
  • Register or login with an Adobe account
  • Enable Hydration, which is used to automatically update the mobile app whenever the app is rebuilt.
